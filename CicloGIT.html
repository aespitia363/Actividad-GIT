FLUJO DE TRABAJO DE GIT

Para entender de mejor manera el flujo de trabajo de GIT, es necesario conocer los tres componentes fundamentales de este, el cual se divide en tres, el primero es el directorio local (working directory), luego tenemos un área temporal (staging área) y por último un repositorio local (repo local), como se muestra en la siguiente imagen.

Ahora bien, en primer lugar el directorio local o de trabajo es cualquier carpeta en la cual se esté trabajando, el área temporal es un espacio donde se enviarán uno o varios archivos de nuestro directorio y allí se podrán se mantendrán todo tipo de cambios que sean realizados desde el directorio. De otra parte, el repositorio local se encargará de guardar todas las versiones de los cambios que se han realizado en los documentos.

Finalmente, el repositorio remoto (remote repo), el cual a diferencia de los otras carpetas, no se encuentra de manera local como su nombre lo indica, este repositorio remoto es una herramienta que manejan diferentes software, los cuales se encargan de guardar en la nube los trabajos del repositorio local de GIT, una herramienta recomendada para esta labor es GITHUB.
